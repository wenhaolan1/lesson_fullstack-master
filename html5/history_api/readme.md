- 传统web开发
    1. 基于HTTP
    2. 每次链接跳转，全新的HTTP请求并且返回内容后，重新渲染页面
    3. 每个文件都是全新的一套， 哪怕有很多重复的地方 导航
    - 缺点？
        1. 文件比较大的， 去服务器走一朝
            速度可能会比较慢，网页的打开速度至关重要 0.1s
        2. 拿了重复的内容，并且做了重复的渲染
        HTTP 协议这个能优化，依赖于HTTP协议去取新的内容(去除重复的东西？nav)
        需要改变的页面地方，动态的去修改
        3. 原页面都会丢掉，等待请求 页面会白屏 内容到后，再显示出来？
